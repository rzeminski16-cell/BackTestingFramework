# Reading Reports

How to interpret the output files generated by backtests and optimisations.

---

## Output Locations

| Analysis Type | Output Directory |
|---|---|
| Single security backtest | `logs/backtests/single_security/` |
| Portfolio backtest | `logs/backtests/portfolio/` |
| Walk-forward optimisation | `logs/optimization_reports/` |
| Vulnerability analysis | `reports/vulnerability/` |
| Per-trade analysis | `reports/per_trade_analysis/` |

---

## Excel Report (`.xlsx`)

The main report file, generated by `ExcelReportGenerator`. Contains multiple sheets:

### Summary Sheet
- All [[Metrics Glossary|performance metrics]] in a single table
- Key figures: total return, Sharpe ratio, max drawdown, win rate, profit factor

### Trade List Sheet
- One row per trade
- Columns: entry date, exit date, entry price, exit price, P/L, P/L%, duration, exit reason

### Equity Curve Sheet
- Chart showing capital value over time
- Drawdown overlay

### Monthly Returns Sheet
- Returns broken down by month and year
- Heatmap visualisation

---

## Trade Log (`.csv`)

A flat CSV with one row per trade. Useful for further analysis in pandas or Excel.

Key columns:

| Column | Description |
|---|---|
| `entry_date` | Date position was opened |
| `exit_date` | Date position was closed |
| `entry_price` | Execution price at entry |
| `exit_price` | Execution price at exit |
| `quantity` | Number of shares |
| `pnl` | Absolute profit/loss (in base currency) |
| `pnl_pct` | Percentage profit/loss |
| `duration_days` | How long the position was held |
| `exit_reason` | Why the trade closed (stop loss, signal, etc.) |

---

## Optimisation Report

Generated by walk-forward optimisation. Contains:

- **Aggregated out-of-sample metrics** — the realistic performance estimate
- **Per-window breakdown** — train metrics vs test metrics for each roll
- **Parameter history** — which parameters were selected in each window
- **Parameter stability** — standard deviation of selected parameters across windows

> [!tip] Reading Parameter Stability
> Low standard deviation = the optimiser consistently finds similar values = robust.
> High standard deviation = parameters vary wildly = likely unstable/overfit.
